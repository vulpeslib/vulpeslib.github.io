Detailed Description

template<typename Mesh>
class vulpes::geometry::ParallelMesh< Mesh >

This class wraps any serial mesh class and adds suitable attribute needed to distributes it in parallel.

There are the following ways to build a parallel mesh:

  • parellize an already build mesh by proc 0 by taking its ownership
  • the map inherited from the base class are local to local
  • there is a copy with already the local to global map
  • the entity mappings are available as both local to global and the inverse

This class wraps a serial mesh class and adds attributes and methods for parallelization.

It enables a mesh, initially created and owned by a single process (rank 0), to be distributed among multiple processes for parallel computation. The parallelization process involves partitioning the mesh and creating local copies for each process, including "ghost" cells that are adjacent to the owned cells but belong to other processes.

Key features:

  • A mesh can be built entirely on a single process (rank 0) and then parallelized.
  • The mesh partitions the entity and keeps a local-to-global and global-to-local maps for each entity
  • The mesh's entity maps (e.g., cell-to-vertex, facet-to-cell) are stored as local-to-local maps. This are the "default" maps.
  • Entity mappings are also available as local-to-global. These are stored as copies, do not require to pass by the entity mappings. Are available with methods called "Global".

◆ initFromRank0()

template<typename Mesh >
void vulpes::geometry::ParallelMesh< Mesh >::initFromRank0 ( std::unique_ptr< Mesh >  rank0_mesh)
inline

Parallelizes a mesh that has already been built on rank 0.

This is the core method for distributing the mesh. It takes a unique pointer to a Mesh object (which is only valid on rank 0) and partitions it using ParMETIS. It then distributes the partitioned data to all processes, builds the ghost cells (which are neighbors of the owned cells but belong to other processes), and constructs the local-to-global and global-to-local maps for all entities (vertices, edges, faces, and cells).

Parameters
rank0_meshA unique pointer to the mesh owned by rank 0. The ownership is moved to this class.

◆ partitionWithParmetis()

template<typename Mesh >
std::vector<::idx_t > vulpes::geometry::ParallelMesh< Mesh >::partitionWithParmetis ( )
protected

Partitions the mesh's dual graph using ParMETIS.

This method uses the ParMETIS library to partition the dual graph of the mesh, where each node represents a cell. It distributes the dual graph data to all processes and calls ParMETIS_V3_PartKway to perform the partitioning. Each process receives its portion of the partitioning result, which indicates which cells it should own.

Returns
A vector of idx_t (ParMETIS's index type) containing the partition assignments for the local cells.

◆ reconstructPartitioningAndScatterDual()

template<typename Mesh >
void vulpes::geometry::ParallelMesh< Mesh >::reconstructPartitioningAndScatterDual ( std::vector< index_t > &  part)
protected

Reconstructs the full mesh partitioning on rank 0 and scatters the (strong) dual graph.

Rank 0 collects the partition results from all other processes, reconstructs the global partitioning, and reorders it to minimize communication overhead. It then slices the global dual graph based on the new partitioning and scatters the relevant parts to each process.

Parameters
partA vector of partition assignments for the local cells.
