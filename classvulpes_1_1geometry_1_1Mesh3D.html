Represents a 3D mesh.

Template Parameters
FloatTypeThe floating-point type for vertex coordinates (e.g., float, double).
IndexTypeThe integer type for indexing vertices, cells, etc. (e.g., int, long).

template<typename FloatType , typename IndexType >
using vulpes::geometry::Mesh3D< FloatType, IndexType >::CellsFaceVtxsUncompressed = std::vector<std::vector<IndexType> >

A vector of vectors, where each inner vector holds the vertex indices for the faces of a cell.

The format is similar to VTU files and voro++, where the first element of each inner vector is the number of vertices for the first face, followed by the vertex indices, and so on for all faces of the cell.

Creates a new, coarser mesh by grouping cells together.

This process, called agglomeration, is used to create a simplified version of the mesh, for example in multigrid solvers.

Parameters
old_to_new_cellA vector mapping each old cell to a new, larger cell ID.
check_connectedIf true, verifies that each new cell is formed by a connected group of old cells.
Returns
A new Derived mesh object representing the agglomerated mesh.

Checks if each new agglomerated cell consists of connected old cells.

It performs a search (DFS) within each group of old cells to ensure they form a single, contiguous block. It will exit with an error if a disconnected group is found.

Builds the edge-related maps.

This method computes and populates the m_edge_to_vtx, m_cell_to_edge, and m_edge_to_cell maps. It does this by iterating through all the faces and their vertices, identifying unique edges, and then determining which cells and vertices are connected to each edge.

Builds the facet-to-cell and facet-to-vertex maps.

This method uses the provided cell-face-vertex connectivity to determine which faces belong to which cells and which vertices belong to each face. It correctly identifies shared faces between cells and builds the m_facet_to_cell and m_facet_to_vtx maps. It relies on the m_cell_to_vtx map being already built.
