Constructs a tensor-product quadrature rule from a 1D quadrature rule.

This function generates a multidimensional quadrature rule by creating the tensor product of a given 1D quadrature rule. The weights are computed as the product of the weights from the 1D rule, and the points are determined by interpreting the multidimensional indices.

Returns
A tensor-product quadrature rule of type QuadratureRule<T, dim>.

◆ getTriangleTabledQuadrature()

const QuadratureRule< mpf_class, 2 > & vulpes::qrule::getTriangleTabledQuadrature ( std::size_t  deg)

Retrieves a precomputed quadrature rule for a triangle.

Get the tabled quadrature rule of selected degree for a triangle.

Provides quadrature rule for the triangles for a specified degree. We employ Witherden and Vincent with 38 digits apart for degree 3 where we employ Hillion I with 17 digits

Parameters
degThe degree of the quadrature rule (up to 20).
Returns
A reference to the precomputed quadrature rule.

We mainly use the Witherden-Vincent rules due to their high numerical precision. Values are computed using Polyquad. Tables are available as supplementary data to the paper link. All the rules are PI.

We tested them with all the monomials, they are accurate up to 38 digits.

F.D: Witherden, P.E. Vincent. the identification of symmetric quadrature rules for finite element methods'. Computers and Mathematics with Applications 69 (2015) 1232-1241. DOI 10.1016/j.camwa.2015.03.017

Since the Witherden-Vincent rules lack a formula of order three, for this specific case we use the (internal) Hillion rule. To ensure the best accuracy we apply few iterations of gradient descent in arbitrary precision to the rule (the loss function is the error on the integrations of the monomials, symmetry of the formula is preserved).

P. Hillion, 'Numerical Integration on a Triangle', International Journal for Numerical Methods in Engineering, Vol. 11, (1977), pp.797-815.

Computes a Gauss-Legendre quadrature rule with arbitrary precision.

Generates (n) nodes and weights for integrating functions on ([-1, 1]) using Gauss-Legendre quadrature, exact for polynomials up to degree (2n-1). Leverages arbitrary-precision arithmetic to ensure accuracy for large (n).

Key Steps:

  • Legendre Polynomials: Computes the Legendre polynomial (P_n(x)) and its derivative (P_n'(x)) exactly using arbitrary precision rationals.
  • Root Finding: Uses Newton's method to refine nodes (roots of (P_n(x))) to the desired precision using arbitray precision floats.
  • Symmetry: Exploits symmetry to compute only half the nodes and weights.
  • Odd (n): Handles the central node separately if (n) is odd.
Parameters
nNumber of quadrature points \(\(n \geq 1\)\).
precisionBit precision for computations (default: 256 bits).
Returns
A gmp 1D quadrature rule with n nodes and weights.

std::size_t  precision = 256

Generates a Gauss-Lobatto quadrature rule in 1D with arbitrary-precision arithmetic.

Gauss-Lobatto quadrature is a variant of Gauss quadrature that includes the endpoints of the interval ([-1, 1]) as nodes. It is exact for polynomials up to degree (2n-3). The nodes and weights are calculated using arbitrary precision.

Key Steps:

  • Legendre Polynomials: Computes the Legendre polynomial (P_n'(x)) and its derivative (P_n''(x)) exactly using arbitrary precision rationals.
  • Root Finding: Uses Newton's method to refine nodes to the desired precision using arbitray precision floats.
  • Symmetry: Exploits symmetry to compute only half the nodes and weights.
  • Odd (n): Handles the central node separately if (n) is odd.
Parameters
nThe number of quadrature points.
precisionThe bit precision for arbitrary-precision arithmetic. Defaults to 256 bits.
Returns
A gmp 1D quadrature rule with n nodes and weights.
