Detailed Description

template<std::size_t Dim, typename E>
class vulpes::fe::FeExpressionFieldComponent< Dim, E >

Extract the field (block) component of a multi-component test/trial expression driven by the integration loop's DOF index.

For a multi-field test function whose evaluate<J,I>() returns a vector of field values, this class extracts entry [I] (test), [J] (trial), or (J,I) (bilinear). The extraction index is NOT a compile-time constant – it is the DOF block index supplied by the integrator at each test/trial DOF, so the result is automatically diagonal across fields.

This is distinct from FeExpressionSpatialComponent, which extracts a fixed compile-time spatial direction (x, y, z) from a vector-output expression.

Typical use: assemble a scalar coefficient against a single field of a multi-component test function without writing separate forms per field:

// phi is a 2-component test function (field 0 = u, field 1 = v).
// fe::field_component(phi) at DOF I=0 returns phi_u, at I=1 returns phi_v.
// The assembled form is block-diagonal: p couples only to its own field.
TestFunction<FS, 0, 2> phi(fe_space);
auto wf = p_proj * fe::field_component(phi);

Compute the polynomial associated to the expression for basis i and j and block bi and bj.

Parameters
fe_valuesFinite element values at quadrature points
jIndex of the trial function basis
iIndex of the test function basis
biBlock index of the trial function
bjBlock index of the test function
Returns
Matrix of polynomials associated to the expression

We use the convention that in the result, each row corresponds to a spatial direction (this extends also to tensors, by unrolling them), while each column corresponds to a addendum of the expression requiring a particular pullback/pushforward transformation. These coefficients are computed by the evaluatePolyCoef method.
