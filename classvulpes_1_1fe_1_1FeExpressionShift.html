Detailed Description

template<std::size_t Dim, typename E>
class vulpes::fe::FeExpressionShift< Dim, E >

Add a constant offset to an FE expression at every quadrature point.

Wraps any expression E and returns m_c + E.evaluate<J,I>(q). The stored constant m_c has the same type as E::OutputType, so the class works uniformly for:

  • Scalar expressions (OutputType = double): offset is a double.
  • Vector expressions (OutputType = Eigen::Vector<double,N>): offset is an Eigen::Vector<double,N> and addition is component-wise.

All update flags, precompute logic, and polynomial order are delegated to the wrapped expression.

For scalar expressions the natural operator+ / operator- syntax is available. For vector expressions use the named factory fe::shift.

Typical use:

auto wf = fe::inner(a_proj + b_const, v); // scalar shift
auto wf = fe::inner(fe::shift(p_vec, p_bg), V); // vector shift

Parameters

jIndex of the trial function basis
iIndex of the test function basis
biBlock index of the trial function
bjBlock index of the test function
Returns
Matrix of polynomials associated to the expression

We use the convention that in the result, each row corresponds to a spatial direction (this extends also to tensors, by unrolling them), while each column corresponds to a addendum of the expression requiring a particular pullback/pushforward transformation. These coefficients are computed by the evaluatePolyCoef method.
