std::pair< AreaIntegral2D< PPoly, FloatType, IndexType >, EdgeIntegral2D< PPoly, FloatType, IndexType > > vulpes::qfree::integrateMonomialsQuadratureFree2D ( const geometry::Cell2D< FloatType > &  cell,
IndexType  deg 
)

Computes the 2D area and edge integrals over a polygonal cell without requiring quadrature points of all monomials up to a fixed degree.

This function calculates the integrals over a 2D polygonal cell for both area and edges up to a given degree. It uses a recursive algorithm based on monomial basis functions, avoiding the need for quadrature points. The result is returned as a pair of AreaIntegral2D and EdgeIntegral2D objects. The template parameter PPoly controls whether the calculations are restricted to polynomial spaces up to a given degree in the sum of exponents of the components.

Template Parameters
PPolyBoolean template parameter indicating whether the polynomial space is truncated. If true, the integrals are calculated for the space \(\mathbb P^k\) (polynomials up to degree deg \( x^i y^j \) where \( i + j \leq \texttt{deg} \) ). If false, it considers all monomials in \( \mathbb Q^k \).
FloatTypeFloating point type (e.g., float, double) used for calculations.
IndexTypeInteger type used for indexing (e.g., int, size_t).
Parameters
[in]cellThe polygonal cell over which the integrals are computed. The vertices of the cell are stored in the vtxs member of the Cell2D object, which is a 2xN matrix where each column is the coordinate of a vertex.
[in]degThe maximum degree of monomials considered for the integrals. The degree is incremented by one for both area and edge integrals in the recursive calculations.
Returns
A std::pair consisting of:
  • AreaIntegral2D: The area integrals of monomials \( x^i y^j\) over the cell, for \( i + j \leq \text{deg} \).
  • EdgeIntegral2D: The edge integrals along each edge of the cell for the same monomials.
Note
: The algorithm loops over each edge (formally each vertex, but in 2D they are practically the same thing) and computes the relative edge integral. We set the origin of the polygon as (0, 0, 0), thus the area integrals can be computed by accumulating the edge integrals (without summing lower area integrals terms)

Computes the volume and lower-dimensional integrals over a 3D element without quadrature.

This function calculates volume integrals (VolumeIntegral3D) and lower-dimensional integrals (LowerIntegral3D) for the given 3D element using an analytical, quadrature-free approach. The calculations involve evaluating integrals over edges and faces of the element using monomial expansions up to a specified degree. The template allows handling different configurations for polynomials of space \( \mathbb{P}^k \) or \( \mathbb{Q}^k \) (PPoly)

Template Parameters
PPolyA boolean template parameter to indicate if the function is using monomials of the space \( \mathbb{P}^k \). If true, the function skips monomials where the sum of exponents is exceeding the degree.
PruningA boolean template parameter to indicate if the edge pruning is applied. In order to apply it, it is necessary to assess the topology of the cell and do suitable pre-computations. TODO: which pre-computations?
FloatTypeThe floating-point type used for computations (e.g., float, double).
IndexTypeThe type used for indexing (typically an integer type).
Parameters
elemA constant reference to the Cell3D<FloatType, IndexType> object, which defines the 3D element, including its vertices (coords), edges (edges), faces (faces), and other structural details.
degThe degree of the polynomial or monomial expansions used for computing the integrals.
Returns
A pair of integrals:
  • VolumeIntegral3D<FloatType>: The integral over the volume of the element.
  • LowerIntegral3D<FloatType>: The integrals over the faces components.
Note
The algorithm is composed by two steps:
  1. Caching of needed quantities (e.g. edge - vertex distances)
  2. Integrals computations

The integrals are computed in a non-recursive manner, starting for the vertexes, then edges, faces and finally volume. The center \( x_0 \) of the entities defining the polyhedron is choosen to be a vertex of the entity itself for edges and faces, and the origin (0, 0, 0) for the volume. The reason is that this allows for pruning of edge integrals in the computation of face integrals. Since the faces integrals are often needed, instead of pruning the face integrals during the computation of the volume integral, it is more convenient to avoid the sum of lower order monomials for the volume. Indeed this approach is a merge of Chin & Sukumar work (which is non-recursive) and the pruning technique proposed by Radley, Houston & Hubbard.

Several other modifications are applied in order to enhance efficiency. For more details see the comments in the source code.

References:

  • Chin, Eric B., Jean B. Lasserre, and N. Sukumar. "Numerical integration of homogeneous functions on convex and nonconvex polygons and polyhedra." Computational Mechanics 56 (2015): 967-981.
  • Antonietti, Paola F., Paul Houston, and Giorgio Pennesi. "Fast numerical integration on polytopic meshes with applications to discontinuous Galerkin finite element methods." Journal of Scientific Computing 77.3 (2018): 1339-1370.
  • Chin, Eric B., and N. Sukumar. "An efficient method to integrate polynomials over polytopes and curved solids." Computer Aided Geometric Design 82 (2020): 101914.
  • Radley, Thomas J., Paul Houston, and Matthew E. Hubbard. "Quadrature-free polytopic discontinuous Galerkin methods for transport problems." arXiv preprint arXiv:2310.10406 (2023).
