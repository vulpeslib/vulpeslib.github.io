Detailed Description

template<std::size_t TestComp, std::size_t TrialComp, std::size_t IndexBegin, std::size_t IndexEnd, std::size_t Dim, typename FeSpace, typename CoeffExpr, typename VExpr>
class vulpes::fe::FeExpressionInnerBlock< TestComp, TrialComp, IndexBegin, IndexEnd, Dim, FeSpace, CoeffExpr, VExpr >

Single-block bilinear expression \( c(x) \, u_l \, v_k \), where \( k \) = TestComp and \( l \) = TrialComp.

Face-side analogue of FeExpressionGradContrBlock for matrix-valued couplings between two components of a vector FE function. Selects exactly one (TestComp, TrialComp) entry of a coefficient matrix at integration time; every other (I,J) block returns void() and is skipped by the integrator.

The test argument v may be either an own-cell FeExpressionFunc or a neighbour-cell FeExpressionNeighborFunc (obtained via fe::neigh). When the latter is supplied the basis values are read from fe_values.neighValues() automatically — the implementation delegates to the sub-expression's evaluate, reusing the same routing that FeExpressionInner already relies on for face neighbour terms.

Typical use case — a numerical flux split for a hyperbolic system with flux Jacobian \( A(n) \):

// Diagonal and neighbour parts of entry (k, l) of A(n) on a face.
auto coeff_kl = (0.5 * c) * n_x * n_x;
auto wf_diag_kl = -fe::inner_block<k, l>(coeff_kl, U, V);
auto wf_neigh_kl = fe::inner_block<k, l>(coeff_kl, U, fe::neigh(V));

Evaluate the (J, I) block contribution at quadrature point q.

J is the trial component (col block) and I is the test component (row block) selected by the integrator's outer loop. The integrator accumulates

row = i_basis * nblocks + I
col = j_basis * nblocks + J

This expression fires only when (J, I) == (TrialComp, TestComp). All other pairs return void().

Returns \( c(x_q) \cdot \phi_i(q) \cdot \phi_j(q) \) for the active pair, where \( \phi_i \) is taken from v (own or neighbour cell depending on VExpr) and \( \phi_j \) from u (own cell). Basis values are obtained by delegating to the sub-expressions' own evaluate so that neighbour routing is preserved.
