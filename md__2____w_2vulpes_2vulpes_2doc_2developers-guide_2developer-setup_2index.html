Developers Setup

  • Install Docker Desktop link and follow the instruction for your OS. To check that the installation is working try to run sudo docker run hello-world
  • Manage Docker as a non-root user link NOTE: if you are on Windows this means opening the terminal with Admin privileges (with right click)
  • Install VS Code link
  • Clone the repository git clone git@github.com:vulpeslib/vulpes.git --recursive NOTE: if you are on Windows, it is much faster to clone in a folder that is owned by the WSL. That is, opening the terminal on WSL and cloning in a directory that is NOT under /mnt
  • Open the repository in VSCode
  • Install the "Docker" and "Dev Container" extension in VS Code (Open the extension panel with Ctrl+Shift+X)
  • Should be promped to enter the .devcontaier when opening the vulpes folder. We you agree the first time VSCode will start building the Docker container automatically. It will take a few minutes.

Every time you install a kernel update you may need to restart your pc to use this setup.

VSCode extensions

Relevant VSCode extensions in the remote container should be automatically installed.

In settings (File > Preferences > Settings) search "Clang_format_style" and set file:/workspaces/vulpes/.clang-format. Then you can format with Ctrl+Shift+I.

Use the Docker from VSCode

  • To create a new terminal from VS Code use Ctrl+Shift+P and select Terminal: Create New Terminal
  • To enter the docker from a generic terminal, find the name of the container using docker ps -a (the image name should be something similar to vsc-vulpes-<hash>), then use docker exec -it <docker container name> /bin/bash to open the container in interactive mode

Simple firsts steps

If you want to jump directly into the code your first step can be something like

mkdir build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j<n_jobs>

To test the code

ctest -j<n_jobs>

To check the failure of a test, you can run the single executable in the tests folder.
