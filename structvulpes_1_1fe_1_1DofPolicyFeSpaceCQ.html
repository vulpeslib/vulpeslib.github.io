Returns the index of the dof in cell enumeration given geometrical information.

Parameters
meshReference to the mesh
cell_idGlobal id of the cell
owning_enty_idGlobal id of the entity (vtx, face, ...) owning the dof
enty_dimDimension of the entity (0 for vtx, 1 for edge, ...)
enty_multiplicityMultiplicity of the dof on the entity (e.g. the second dof on an edge has multiplicity 1)
Returns
The index of the dof in cell enumeration

We write each degree of freedom as a convex combination of the vertices of the hyper-cube that define the cell. Then, recover the local id of the dof by searching in a precomputed table. E.g., for the faces, we recover the global vertexes that define the face and write it as a convex combination of them. In this way we have a unique representation of the dof that does not depend on the frame of reference. The global vtx id of the face are face_to_vtx_map[owning_enty_id_local]. Their coefficient are m_face_dofs[enty_multiplicity] (up to a rescaling factor, since the rescaling is order^Dim and m_face_dofs are computed in 2D). Then, we can write this convex combination with respect to the local dofs of the cells since it has all the four vertices needed. Thus we find which are the position of these four global vertices in local enumeration (of the 8 vertices that define the cell), so we can write the dof as a ConvexCartIndex

Warning
The convention for vertex numbering of the hexahedron in a cell is: (v0, v1, v5, v4) Front (v3, v2, v6, v7) Back (v0, v3, v7, v4) Left (v1, v2, v6, v5) Right (v0, v1, v2, v3) Bottom (v4, v5, v6, v7) Top
